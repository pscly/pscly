## 关于我

我主要在做一类事情：**把 AI 接进现有工具链、系统和流程里，做成真正能落地的自动化能力。**

相比只做一个模型调用 Demo，我更关注这些问题：

- 如何把 AI 接进采集、清洗、分类、通知、审核、生成等具体流程
- 如何让模型能调用工具、访问服务、串联 API，而不是孤立输出文本
- 如何把零散脚本、接口和任务编排成可维护、可复用、可部署的系统
- 如何让自动化方案在真实环境中持续运行，而不是一次性实验

## 我擅长的方向

| 方向 | 关注重点 | 常见落地方式 |
| --- | --- | --- |
| AI 工作流自动化 | 让重复流程变成稳定能力 | Python 脚本、任务编排、消息通知、自动处理链路 |
| AI 集成 | 让模型真正接入系统与工具 | API 集成、Tool Calling、服务联动、内部工具接入 |
| 自动化工程 | 把临时脚本做成可持续交付的能力 | FastAPI、Docker、Linux 服务化、部署与运维 |
| 效率工具建设 | 减少重复操作与人工切换成本 | CLI、浏览器自动化、批处理、流程封装 |

## 代表实践与公开项目

> 这里放的是更能代表我当前能力方向的公开工程与实践主题。重点不是仓库数量，而是能否体现自动化、系统集成与工程落地能力。

### 公开项目

- [`xinliuend`](https://github.com/pscly/xinliuend) - 一个更完整的服务端工程实践，包含 FastAPI 后端、同步能力、Memos 集成、Docker 部署、管理后台与配套文档化说明。

### 实践方向
